📦
Batch Reads

Read contiguous register blocks in a single FC03 transaction (up to 125 registers). The block plan tab defines all optimized read groups.

⏱
Data Prioritization

Poll critical registers (power, breaker, alarms) every 1 s. Slow-changing data (energy, hours, I/O) every 5–30 s. Use User Alarm Groups (regs 101–110) for event-driven alerting.

🔗
Persistent Connections

Keep TCP connections open. Do not create a new socket per transaction. ThingsBoard Modbus connector handles this automatically.

🔄
Retry with Limits

Implement retries for failed reads. Limit to 2–3 attempts with 1000–3000 ms timeouts. Mark stale data if all retries fail.

📐
Two-Word (32-bit) Registers

Power and energy registers are 32-bit (2 words). MSW is at the lower address. Combine as: value = (MSW << 16) | LSW, then apply scaling and offset.

🔧
ThingsBoard Modbus Connector Config

In tb-gateway Modbus connector, use "type": "holding", "functionCode": 3. For 32-bit: set "objectsCount": 2 and "byteOrder": "BIG". Map each register to a telemetry key matching the block plan above.

🏷
Suggested Telemetry Key Names

Use consistent keys for dashboard binding: gen_avg_voltage, gen_avg_current, gen_frequency, gen_real_power_kw, gen_pf, engine_speed_rpm, oil_pressure_psi, coolant_temp_f, fuel_level_pct, breaker_status, operating_mode, engine_hours.

⚠️
Write Operations (Port 50200)

Use FC16 (Write Multiple Registers) on port 50200. NEVER enable write access without proper authentication and operator-level permissions. Write commands include alarm acknowledge, mode control, start/stop, and setpoint adjustments.
